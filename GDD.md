# 버블 점프 (Bubble Jump) — Game Design Document

## 1. 개요

| 항목 | 값 |
|------|-----|
| 장르 | 2D 싱글 스크린 아케이드 플랫포머 |
| 엔진 | Phaser 3 (WebGL, Canvas fallback) |
| 타겟 해상도 | 480×720 (세로형, 모바일 portrait) |
| 프레임 | 60fps |
| 조작 | 터치 (모바일), 키보드 (데스크톱 폴백) |
| 플레이 시간 | 1회 평균 60~120초 |
| 목표 | 웨이브를 생존하며 적을 밟아 처치하고 최고 점수 달성 |

---

## 2. 화면 구성

### 2.1 게임 월드

```
┌──────────────────────────┐
│  HUD (점수, 체력, 웨이브) │  y=0~40
├──────────────────────────┤
│                          │
│     ┌──────────┐         │  플랫폼 3 (y=200)
│     └──────────┘         │
│                          │
│  ┌──────────┐            │  플랫폼 2 (y=380)
│            └──────────┐  │
│                          │
│     ┌──────────┐         │  플랫폼 1 (y=560)
│     └──────────┘         │
│                          │
│██████████████████████████│  바닥 (y=680)
└──────────────────────────┘
```

- **바닥**: x=0~480, y=680. 화면 하단 전체를 덮는 지형. `terrain_grass_block` 타일로 구성.
- **플랫폼**: 가로 192px(타일 3개) 크기의 발판. `terrain_grass_horizontal_left/middle/right` 사용. 상단에서만 충돌 (one-way platform).
- **플랫폼 배치**: 3개 고정. 아래에서부터 y=560, y=380, y=200. 각 플랫폼의 x 위치는 웨이브마다 랜덤 배치하되, 플레이어가 점프로 도달 가능한 범위 내에 위치한다 (수평 간격 최대 192px, 수직 간격 180px 이내).
- **Wrap-around**: 캐릭터와 적이 화면 왼쪽 끝(x<0)을 넘으면 오른쪽 끝(x=480)에서 나오고, 반대도 동일. 시각적으로 끊김 없이 동일 오브젝트가 양쪽에 동시에 보여야 한다.

### 2.2 배경

3레이어 정적 배경 (스크롤 없음, 타일링 배치):
1. 최하: `background_solid_sky` — 화면 전체 채움
2. 중간: `background_fade_hills` — y=300 아래 배치, 알파 0.5
3. 전경: `background_color_hills` — y=500 아래 배치, 알파 0.7

## 3. 플레이어

### 3.1 스펙

| 속성 | 값 |
|------|-----|
| 스프라이트 | `characters` 아틀라스, `character_beige_*` |
| 크기 | 48×48px (128px 원본을 0.375 스케일) |
| 물리 바디 | 40×44px 직사각형 (좌우 4px, 하단 4px 여백) |
| 이동 속도 | 200px/s |
| 점프 속도 | -420px/s (위 방향 초기 속도) |
| 중력 | 900px/s² |
| 최대 낙하 속도 | 500px/s |
| 적 밟기 바운스 | -350px/s (위 방향) |
| 체력 | 3 (하트 3개) |
| 피격 무적 시간 | 1.5초 |
| 시작 위치 | x=240, y=640 (바닥 위 중앙) |

### 3.2 애니메이션

| 상태 | 프레임 | fps |
|------|--------|-----|
| 대기 | `character_beige_idle` | - (단일) |
| 걷기 | `character_beige_walk_a`, `_walk_b` | 8fps 루프 |
| 점프(상승) | `character_beige_jump` | - (단일) |
| 낙하 | `character_beige_front` | - (단일) |
| 피격 | `character_beige_hit` | - (단일, 1.5초간 유지) |

### 3.3 상태 전이

```
대기 ─(좌/우 입력)─→ 걷기
대기 ─(점프 입력)──→ 점프
걷기 ─(입력 해제)──→ 대기
걷기 ─(점프 입력)──→ 점프
점프 ─(vy≥0)──────→ 낙하
낙하 ─(지면 착지)──→ 대기 or 걷기
ANY ──(피격)──────→ 피격 → 1.5초 후 → 이전 상태
```

### 3.4 이동 동작

- **지상 이동**: 좌/우 입력 시 즉시 해당 방향으로 200px/s. 입력 해제 시 즉시 정지 (관성 없음).
- **공중 이동**: 점프/낙하 중에도 좌/우 이동 가능. 동일 속도 200px/s.
- **점프**: 지면에 발이 닿은 상태에서만 점프 가능. 1단 점프만 존재.
- **방향 전환**: 이동 방향에 따라 스프라이트 flipX. 오른쪽이 기본(flipX=false).

## 4. 적 (Enemy)

### 4.1 공통 규칙

- 모든 적은 화면 양쪽 가장자리에서 스폰된다. 왼쪽(x=-32) 또는 오른쪽(x=512)에서 등장.
- 모든 적은 wrap-around 적용.
- 적끼리는 충돌하지 않는다 (서로 통과).
- 적은 플랫폼 위를 걸어다니거나, 공중을 떠다닌다.

### 4.2 처치 판정

```
플레이어.하단 바디 경계 < 적.상단 바디 경계 + 8px
AND 플레이어.vy > 0 (낙하 중)
→ 적 처치, 플레이어 바운스(-350px/s)
```

위 조건이 아닌 상태에서 플레이어와 적의 바디가 겹치면 → **피격**.

### 4.3 적 종류

#### 지상 적

| 적 | 스프라이트 키 | 크기 | 이동속도 | 행동 | 점수 |
|----|-------------|------|---------|------|------|
| 슬라임 | `slime_normal_*` | 40×32px | 60px/s | 플랫폼 위를 좌우 왕복. 끝에 도달하면 반전. | 100 |
| 벌레 | `worm_normal_*` | 40×28px | 40px/s | 슬라임과 동일하나 더 느림. | 50 |
| 쥐 | `mouse_*` | 44×32px | 120px/s | 빠른 좌우 왕복. | 150 |
| 무당벌레 | `ladybug_*` | 40×32px | 80px/s | 좌우 왕복. 밟으면 뒤집혀서 미끄러짐(shell 상태 없음, 즉시 처치). | 100 |
| 달팽이 | `snail_*` | 44×32px | 30px/s | 매우 느린 이동. 1회 밟으면 껍질(`snail_shell`)만 남고, 껍질을 다시 밟으면 처치. | 1차 50, 2차 200 |

#### 공중 적

| 적 | 스프라이트 키 | 크기 | 이동속도 | 행동 | 점수 |
|----|-------------|------|---------|------|------|
| 벌 | `bee_*` | 36×36px | 70px/s | 수평 직선 이동. 사인파 상하 진동(진폭 30px, 주기 2초). | 200 |
| 파리 | `fly_*` | 36×36px | 100px/s | 수평 직선 이동. 직선 경로. | 150 |

#### 밟기 불가 (회피 전용)

| 적 | 스프라이트 키 | 크기 | 이동속도 | 행동 | 접촉 시 |
|----|-------------|------|---------|------|---------|
| 톱날 | `saw_*` (enemies) | 44×44px | 50px/s | 플랫폼 가장자리를 따라 이동 또는 고정 경로 순환. | 어느 방향이든 접촉 시 피격 |
| 가시슬라임 | `slime_spike_*` | 40×36px | 50px/s | 지상 좌우 왕복. | 어느 방향이든 접촉 시 피격 (밟기 포함) |

### 4.4 적 애니메이션

모든 지상 적: `_walk_a`, `_walk_b` 교대 재생 8fps.
모든 공중 적: `_a`, `_b` 교대 재생 10fps.
처치 시: `_rest` 프레임 표시 → 0.3초간 아래로 낙하하며 페이드아웃(알파 0→제거). 동시에 처치 위치에서 점수 텍스트 팝업(위로 떠오르며 1초 후 소멸).

### 4.5 적 스폰

- 스폰 위치: 화면 왼쪽 바깥(x=-32) 또는 오른쪽 바깥(x=512). 랜덤 선택.
- 스폰 y 위치: 지상 적은 바닥 또는 플랫폼 위. 공중 적은 y=100~500 랜덤.
- 스폰 방향: 왼쪽에서 스폰 시 오른쪽으로 이동, 오른쪽에서 스폰 시 왼쪽으로 이동.

## 5. 아이템

### 5.1 아이템 블록

- 화면 내 고정 위치에 `block_exclamation_active` 타일 1개 배치. 위치: x=240, y=290 (플랫폼 3 아래).
- 플레이어가 아래에서 머리로 치면 아이템 출현. 블록은 `block_exclamation` (비활성)으로 변경.
- 웨이브 시작 시 블록은 다시 `block_exclamation_active`로 리셋.
- 블록 크기: 48×48px. 물리 바디: 정적(static), 전방향 충돌.

### 5.2 아이템 목록

| 아이템 | 스프라이트 | 효과 | 출현 확률 | 지속 시간 |
|--------|-----------|------|----------|----------|
| 코인 | `coin_gold` | 점수 +500 | 50% | 즉시 |
| 하트 | `heart` | 체력 +1 (최대 3) | 20% | 즉시 |
| 별 | `star` | 무적 상태. 접촉만으로 적 처치 가능. 플레이어 깜빡임(알파 0.5↔1.0, 주기 0.1초). | 15% | 5초 |
| 스프링 | `spring` | 다음 1회 점프 높이 2배 (-840px/s). 플레이어 발 밑에 스프링 이펙트. | 15% | 다음 1회 점프 |

아이템은 블록 위(y - 52px)에서 출현하며 작은 포물선으로 위로 튀어오른 뒤 낙하한다 (vy=-200, 중력 적용). 바닥/플랫폼에 착지 후 정지. 출현 후 8초 내 미획득 시 깜빡이다(6초 시점부터) 소멸.

## 6. 웨이브 시스템

### 6.1 구조

한 웨이브 = 정해진 수의 적이 모두 스폰되고, 모두 처치되면 종료.

```
웨이브 시작
  → 적 순차 스폰 (스폰 간격에 따라)
  → 모든 적 처치 시 웨이브 클리어
  → "WAVE N CLEAR!" 텍스트 1.5초 표시
  → 웨이브 보너스 점수 지급
  → 다음 웨이브 시작
```

### 6.2 웨이브 테이블

| 웨이브 | 적 수 | 스폰 간격 | 적 구성 | 이동속도 배율 |
|--------|------|----------|---------|-------------|
| 1 | 5 | 2.0초 | 벌레 ×3, 슬라임 ×2 | 1.0 |
| 2 | 6 | 1.8초 | 슬라임 ×3, 벌레 ×2, 무당벌레 ×1 | 1.0 |
| 3 | 7 | 1.6초 | 슬라임 ×2, 무당벌레 ×2, 벌 ×2, 달팽이 ×1 | 1.0 |
| 4 | 8 | 1.5초 | 쥐 ×2, 벌 ×2, 파리 ×2, 슬라임 ×2 | 1.1 |
| 5 | 9 | 1.3초 | 쥐 ×2, 파리 ×2, 톱날 ×1, 슬라임 ×2, 벌 ×2 | 1.1 |
| 6 | 10 | 1.2초 | 가시슬라임 ×2, 톱날 ×1, 쥐 ×2, 벌 ×3, 파리 ×2 | 1.2 |
| 7+ | 10+N | 1.0초 | 전 종류 랜덤 (톱날/가시슬라임 최대 2마리) | 1.0 + N×0.05 (최대 2.0) |

- 이동속도 배율: 각 적의 기본 속도에 곱한다.
- 웨이브 7 이후는 무한 반복. 적 수는 웨이브당 +1, 속도 배율은 +0.05씩 증가.

### 6.3 웨이브 클리어 보너스

- 기본 보너스: 웨이브 번호 × 300점
- 노데미지 클리어 보너스: 해당 웨이브에서 피격 0회 시 추가 +500점

## 7. 콤보 시스템

### 7.1 정의

- **콤보**: 지면에 착지하지 않고 연속으로 적을 밟아 처치한 횟수.
- 적을 밟으면 바운스하므로 착지 없이 다음 적을 밟을 수 있다.

### 7.2 점수 배율

| 콤보 수 | 배율 |
|---------|------|
| 1 (단일 처치) | ×1 |
| 2 | ×2 |
| 3 | ×3 |
| 4 | ×5 |
| 5+ | ×8 |

- 적 처치 시 획득 점수 = 해당 적 기본 점수 × 콤보 배율
- 지면(바닥 또는 플랫폼)에 착지하면 콤보 리셋.

### 7.3 콤보 연출

- 콤보 2 이상부터 화면 중앙 상단에 "×2 COMBO!" 텍스트 표시. 스케일 1.5→1.0 트윈(0.2초).
- 콤보 수에 따라 텍스트 색상 변경: 2=흰색, 3=노랑, 4=주황, 5+=빨강.
- 콤보 리셋 시 텍스트 페이드아웃(0.3초).

## 8. 피격과 게임오버

### 8.1 피격

1. 체력 -1
2. 플레이어 넉백: 적의 반대 방향으로 vx=±150px/s, vy=-250px/s
3. 피격 애니메이션 (`_hit`) 재생
4. 1.5초간 무적 (스프라이트 알파 0.3↔1.0 깜빡임, 주기 0.1초)
5. 무적 종료 후 정상 복귀

### 8.2 게임오버

1. 체력 0 도달
2. 플레이어 피격 애니메이션 → 1초간 정지
3. "GAME OVER" 텍스트 화면 중앙 표시 (스케일 0→1 트윈, 0.5초)
4. 최종 점수, 최고 웨이브 표시
5. 2초 후 "TAP TO RESTART" 표시
6. 탭 시 게임 씬 재시작

### 8.3 점수 저장

- `localStorage`에 `bubbleJump_highScore` 키로 최고 점수 저장.
- 게임오버 화면에서 현재 점수가 최고 점수 초과 시 "NEW BEST!" 텍스트 표시.

## 9. HUD

화면 상단(y=4~36) 고정 UI. 깊이(depth) 최상위.

```
┌──────────────────────────────┐
│ ♥♥♡   WAVE 3   ★ 12,500  │
└──────────────────────────────┘
```

| 위치 | 요소 | 구현 |
|------|------|------|
| 좌측 (x=8) | 체력 | `hud_heart` × 현재 체력 + `hud_heart_empty` × (3-현재 체력). 크기 24×24, 간격 4px |
| 중앙 (x=240, anchor 중앙) | 웨이브 | Phaser BitmapText 또는 일반 Text. "WAVE N" 흰색 16px |
| 우측 (x=472, anchor 우측) | 점수 | `hud_coin` 아이콘 + 점수 숫자. 흰색 16px. 점수 변경 시 스케일 1.2→1.0 트윈(0.15초) |

## 10. 씬 구성

### 10.1 BootScene

- Phaser 설정 초기화
- LoadScene으로 전환

### 10.2 LoadScene

- 모든 에셋 로드 (아틀라스 4종 + 사운드 10종)
- 로딩 바 표시: 화면 중앙, 가로 300px 흰색 테두리 바. 내부 초록색 채움.
- 로드 완료 시 MenuScene으로 전환

### 10.3 MenuScene

```
┌──────────────────────────┐
│                          │
│                          │
│      🫧 BUBBLE JUMP      │  타이틀 (흰색 32px bold)
│                          │
│     [ TAP TO START ]     │  깜빡임 (알파 0↔1, 주기 1초)
│                          │
│      BEST: 12,500        │  최고 점수 (흰색 16px)
│                          │
│   캐릭터 idle 애니메이션   │  장식용 캐릭터 (화면 하단 중앙)
│██████████████████████████│
└──────────────────────────┘
```

- 배경: GameScene과 동일한 3레이어 배경
- 탭 또는 아무 키 입력 시 GameScene으로 전환

### 10.4 GameScene

- 메인 게임 로직 전체 담당
- 게임오버 시 자체 UI 오버레이로 처리 (씬 전환 없음)
- 재시작 시 `this.scene.restart()`

## 11. 조작

### 11.1 모바일 (터치)

화면 하단에 반투명 가상 버튼:

```
┌──────────────────────────┐
│          (게임 영역)       │
│                          │
│  [←] [→]          [⬆]   │  y=640~720
└──────────────────────────┘
```

| 버튼 | 위치 | 크기 | 동작 |
|------|------|------|------|
| ← | x=8, y=648 | 64×64 | 왼쪽 이동. 터치 중 지속 |
| → | x=80, y=648 | 64×64 | 오른쪽 이동. 터치 중 지속 |
| 점프 | x=408, y=648 | 64×64 | 점프. 터치 시 1회 |

- 버튼 비주얼: 원형, 흰색, 알파 0.3. 터치 중 알파 0.6.
- 멀티터치 지원: 이동 + 점프 동시 입력 가능.

## 13. 물리 설정

Phaser Arcade Physics 사용.

```js
physics: {
    default: 'arcade',
    arcade: {
        gravity: { y: 900 },
        debug: false
    }
}
```

| 그룹 | 타입 | 충돌 대상 |
|------|------|----------|
| player | dynamic | platforms, ground, enemies, items, itemBlock |
| platforms | static | player, enemies(지상) |
| ground | static | player, enemies(지상), items |
| enemies | dynamic | 없음 (overlap으로 처리) |
| items | dynamic | platforms, ground |
| itemBlock | static | player |

- player ↔ enemies: `overlap` 으로 감지 후 밟기/피격 분기 처리.
- player ↔ items: `overlap` 으로 감지 후 아이템 효과 적용.
- player ↔ itemBlock: `collider`로 충돌. 플레이어가 아래에서 블록 하단에 부딪히면 (`player.body.touching.up && itemBlock.body.touching.down`) 아이템 출현.

## 17. 핵심 규칙 체크리스트

구현 완료 후 다음 항목을 모두 확인한다:

- [ ] 캐릭터가 화면 왼쪽 끝을 넘으면 오른쪽에서 나온다 (반대도 동일)
- [ ] 적도 동일하게 wrap-around 된다
- [ ] 적을 위에서 밟으면 처치되고 바운스한다
- [ ] 적을 옆/아래에서 접촉하면 피격당한다
- [ ] 톱날/가시슬라임은 어떤 방향으로 접촉해도 피격이다
- [ ] 피격 후 1.5초 무적이며 깜빡인다
- [ ] 콤보 배율이 정확히 적용된다
- [ ] 착지 시 콤보가 리셋된다
- [ ] 아이템 블록을 아래에서 치면 아이템이 나온다
- [ ] 별 무적 중 접촉만으로 적이 처치된다
- [ ] 별 무적 중 밟기 불가 적(톱날/가시슬라임)도 처치된다
- [ ] 웨이브 전멸 시 보너스 점수와 다음 웨이브가 진행된다
- [ ] 체력 0이면 게임오버된다
- [ ] 최고 점수가 localStorage에 저장된다
- [ ] 모바일 터치와 키보드 동시 지원된다
- [ ] 플랫폼은 위에서만 착지 가능하다 (아래서 통과)
